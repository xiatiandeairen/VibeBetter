# AI Coding 工作流文档体系

## 概述

本目录定义了 VibeBetter 项目的 AI Coding 协作工作流，包含需求管理、任务跟踪和反馈闭环三个核心流程。

## 文档结构

```
docs/workflow/
├── README.md                    # 本文件：工作流说明
├── scoring-standards.md         # 多维度评分标准
├── iterations/                  # 迭代归档
│   ├── iteration-001.md         # 第一轮迭代
│   └── ...
└── templates/
    ├── task-template.md         # 任务文档模板
    └── feedback-template.md     # 反馈文档模板
```

## 核心工作流

```
需求输入 → 任务拆分 → 执行 & 记录 → 验收 & 评分 → 反馈 & 优化
   │          │           │              │              │
   ▼          ▼           ▼              ▼              ▼
 需求文档    任务文档     进度更新      评分报告       反馈文档
                         风险记录                    CLAUDE.md 更新
```

### 阶段 1：需求输入
- 用户提供需求描述（自然语言 / 结构化文档）
- AI Agent 解析需求，确认范围和验收标准

### 阶段 2：任务拆分
- AI Agent 将需求拆分为可执行的任务清单
- 每个任务包含：目标、验证清单、预估工期、依赖关系
- 输出：任务文档（使用 `templates/task-template.md` 模板）

### 阶段 3：执行 & 记录
- AI Agent 按任务清单执行
- 实时记录：完成状态、遇到的问题、技术决策、风险点
- 每个任务完成后更新进度

### 阶段 4：验收 & 评分
- 用户验收任务文档，确认工作与预期一致
- 使用 `scoring-standards.md` 进行多维度评分
- 输出差距分析和改进建议

### 阶段 5：反馈 & 优化
- 用户输出反馈建议
- AI Agent 生成反馈文档，提取优化点
- 更新 `CLAUDE.md` 和 `project.md` 中的协作规则
- 归档本轮迭代
