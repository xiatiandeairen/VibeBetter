# AI Coding 自驱动流程 v1

## 核心循环

```
分析项目 → 找到需求 → 开发需求 → 验证需求 → 成功提交/失败回滚 → 总结优化 → 下一循环
```

## 每个循环的检查清单

### 1. 分析（5 min）
- [ ] 当前项目状态是什么？
- [ ] 用户核心痛点解决了吗？（决策依据 + 效果度量）
- [ ] 哪个需求的 ROI 最高？

### 2. 需求确认（3 min）
- [ ] 这个需求是正确方向吗？（服务于核心痛点）
- [ ] ROI 分析：投入多少时间？产出什么价值？
- [ ] 范围是否足够小且可验证？

### 3. 开发（主要时间）
- [ ] 按功能维度拆分 commit
- [ ] 每个 commit 是一个可回滚的检查点
- [ ] 代码质量：lint + test 通过

### 4. 验证
- [ ] lint 通过
- [ ] test 通过
- [ ] build 成功
- [ ] 功能可演示

### 5. 提交/回滚
- 成功 → commit + push + 记录
- 失败 → 回滚到上一个检查点，记录失败原因
- 多次失败 → 放弃，记录原因，开始下一个需求

### 6. 总结优化（5 min）
- [ ] 本轮做了什么？效果如何？
- [ ] 流程中哪里可以优化？
- [ ] 更新自驱动流程文档

## ROI 评估框架

```
ROI = 用户价值 × 使用频率 / 开发成本

高 ROI：核心闭环功能、修复阻塞用户的 bug
中 ROI：体验优化、性能提升
低 ROI：非核心功能、过度抽象
```

## 回滚策略

- 每个功能一个 commit，便于 `git revert`
- 重大变更前创建 git tag 作为检查点
- 数据库变更与代码变更分开提交
